# Conformance failure list: tests we don't pass yet.
#
# This single file is shared by both conformance workflows:
#   * the regular CI job builds protoc from the pinned Protobuf
#     dependency (see mix.exs) and runs `mix conformance_test --profile pinned`
#   * the scheduled job builds protoc from Protobuf's
#     `main` branch and runs `mix conformance_test --profile latest`
#
# Newer protoc versions add/retire conformance tests and occasionally change a test's expected
# result, so some entries apply to only one of those protoc versions. The conformance runner
# treats a failure-list entry that *succeeds* or that *matches no test* as a fatal error, so each
# run must see exactly the entries that fail for its protoc version. Tag the version-specific
# lines with a trailing comment so `mix conformance_test` can filter them:
#
#   * `# only-on: latest`  -> kept only for `--profile latest`
#   * `# only-on: pinned`  -> kept only for `--profile pinned`
#
# Untagged lines apply to both profiles. Remove entries as the underlying issues are fixed.

# Rejecting JSON objects with duplicate keys (including the same field spelled in two different
# casings, e.g. "optionalInt32" and "optional_int32") requires a JSON parser that surfaces

# duplicates; Protobuf's `main` requires them to be rejected. These are "Recommended".
